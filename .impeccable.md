# Design System

## Design Context

### Users

企業內部員工，在日常工作中使用此 RAG 應用查詢知識庫、搜尋內部文件、獲得工作輔助。使用情境通常是：

- 快速查找公司政策、流程文件
- 詢問技術或業務相關問題
- 需要專注且高效的工作輔助工具

### Brand Personality

**三個關鍵詞**：簡約、高效、可靠

- **語氣**：專業但不冷漠，直接但不粗魯
- **情感目標**：讓使用者感到工具值得信賴、能快速完成任務
- **互動風格**：最少點擊、最快到達答案

### Aesthetic Direction

**視覺基調**：純黑白極簡主義

- **配色**：僅使用黑、白、灰階（neutral palette）
- **無彩色強調**：不使用 primary color，改用字重、大小、對比度區分層級
- **參考**：ChatGPT / Claude 的對話式介面——乾淨的訊息流、簡潔的輸入區、專注於內容
- **Anti-reference**：避免花俏的漸層、彩色按鈕、過多裝飾元素

**主題**：跟隨系統偏好（自動 light/dark mode）

### Design Principles

1. **內容優先**：UI 應該隱形，讓使用者專注於對話內容和搜尋結果
2. **一眼可辨**：資訊層級清晰，重要操作一目了然
3. **最短路徑**：減少點擊次數，常用功能觸手可及
4. **一致性**：相同功能用相同樣式，不製造認知負擔
5. **無障礙優先**：高對比度黑白配色天然利於閱讀

## Color Tokens

基於 Nuxt UI / Tailwind 4，使用純黑白作為 primary：

```css
/* app/assets/css/main.css */
:root {
  --ui-radius: 0.375rem;
  --ui-primary: black;
}

.dark {
  --ui-primary: white;
}
```

- Light mode：primary 為純黑
- Dark mode：primary 為純白
- 圓角統一 6px (0.375rem)

**語意色彩**（僅用於系統回饋）：

| 用途 | 顏色        | 使用時機               |
| ---- | ----------- | ---------------------- |
| 錯誤 | red-500     | 表單驗證、系統錯誤     |
| 警告 | amber-500   | 重要提醒、風險操作確認 |
| 成功 | green-500   | 操作完成、儲存成功     |
| 資訊 | neutral-500 | 一般提示               |

**重要**：語意色彩僅用於系統回饋（alert、toast），不用於裝飾或強調。

## Nuxt UI 語意色彩（嚴格約束）

Nuxt UI 內建 color mode 處理，**禁止**自行指定 Tailwind 色彩，否則 light/dark 切換會失效。

### 必須使用語意類別

| 用途       | 語意類別           | 禁止                                       |
| ---------- | ------------------ | ------------------------------------------ |
| 主要文字   | `text-default`     | ❌ `text-black`, `text-neutral-900`        |
| 次要文字   | `text-muted`       | ❌ `text-gray-500`, `text-neutral-500`     |
| 第三層文字 | `text-toned`       | ❌ `text-gray-600`, `text-neutral-600`     |
| 淡化文字   | `text-dimmed`      | ❌ `text-gray-400`, `text-neutral-400`     |
| 強調文字   | `text-highlighted` | ❌ `text-black`, `text-neutral-900`        |
| 反轉文字   | `text-inverted`    | ❌ `text-white`                            |
| 預設背景   | `bg-default`       | ❌ `bg-white`, `bg-neutral-50`             |
| 淡化背景   | `bg-muted`         | ❌ `bg-gray-50`, `bg-neutral-100`          |
| 凸起背景   | `bg-elevated`      | ❌ `bg-white`                              |
| 強調背景   | `bg-accented`      | ❌ `bg-gray-100`                           |
| 反轉背景   | `bg-inverted`      | ❌ `bg-black`, `bg-neutral-900`            |
| 預設邊框   | `border-default`   | ❌ `border-gray-200`, `border-neutral-200` |
| 淡化邊框   | `border-muted`     | ❌ `border-gray-100`                       |

### 禁止事項

1. **禁止硬編碼色彩**：`text-gray-*`, `text-neutral-*`, `bg-gray-*`, `bg-neutral-*`
2. **禁止 `dark:` prefix**：Nuxt UI 會自動處理，自己寫 `dark:text-white` 會衝突
3. **禁止 CSS 變數以外的黑白**：`text-black`, `text-white`, `bg-black`, `bg-white`

### 允許例外

- `--ui-primary: black` / `--ui-primary: white` 在 CSS 變數中（非 class）
- Nuxt UI 元件的 `color` prop：`color="neutral"`, `color="error"` 等

## Typography

使用 DM Sans 作為主要字體：

```css
@theme {
  --font-sans: 'DM Sans', sans-serif;
}
```

DM Sans 特性：

- 幾何無襯線字體，清晰現代
- 良好的中英文混排效果
- Nuxt Fonts 自動載入

**層級區分**（用字重而非顏色）：

| 層級 | 字重 | 大小 | 用途               |
| ---- | ---- | ---- | ------------------ |
| H1   | 700  | 2xl  | 頁面標題           |
| H2   | 600  | xl   | 區塊標題           |
| H3   | 600  | lg   | 子區塊標題         |
| Body | 400  | base | 內文               |
| Meta | 400  | sm   | 次要資訊、時間戳記 |
| Code | 400  | sm   | 程式碼、技術名詞   |

## Component Patterns

### 對話式介面（Nuxt UI Chat 組件）

使用 Nuxt UI 的 chat 組件系列搭配 Vercel AI SDK：

**基本結構**

```vue
<script setup lang="ts">
  import { isReasoningUIPart, isTextUIPart, isToolUIPart, getToolName } from 'ai'
  import { Chat } from '@ai-sdk/vue'
  import { isPartStreaming, isToolStreaming } from '@nuxt/ui/utils/ai'

  const input = ref('')
  const chat = new Chat({
    onError: (error) => console.error(error),
  })

  function onSubmit() {
    if (!input.value.trim()) return
    chat.sendMessage({ text: input.value })
    input.value = ''
  }
</script>

<template>
  <UDashboardPanel>
    <template #body>
      <UContainer>
        <UChatMessages :messages="chat.messages" :status="chat.status">
          <template #content="{ message }">
            <template
              v-for="(part, index) in message.parts"
              :key="`${message.id}-${part.type}-${index}`"
            >
              <!-- 思考過程 -->
              <UChatReasoning
                v-if="isReasoningUIPart(part)"
                :text="part.text"
                :streaming="isPartStreaming(part)"
              />

              <!-- 工具調用 -->
              <UChatTool
                v-else-if="isToolUIPart(part)"
                :text="getToolName(part)"
                :streaming="isToolStreaming(part)"
              />

              <!-- 文字內容 -->
              <template v-else-if="isTextUIPart(part)">
                <MDC
                  v-if="message.role === 'assistant'"
                  :value="part.text"
                  :cache-key="`${message.id}-${index}`"
                  class="*:first:mt-0 *:last:mb-0"
                />
                <p v-else class="whitespace-pre-wrap">{{ part.text }}</p>
              </template>
            </template>
          </template>
        </UChatMessages>
      </UContainer>
    </template>

    <template #footer>
      <UContainer class="pb-4 sm:pb-6">
        <UChatPrompt v-model="input" :error="chat.error" @submit="onSubmit">
          <UChatPromptSubmit
            :status="chat.status"
            @stop="chat.stop()"
            @reload="chat.regenerate()"
          />
        </UChatPrompt>
      </UContainer>
    </template>
  </UDashboardPanel>
</template>
```

**Chat 組件說明**

| 組件              | 用途                                |
| ----------------- | ----------------------------------- |
| UChatMessages     | 可捲動訊息列表，自動捲動            |
| UChatReasoning    | 可摺疊思考區塊，串流中自動展開      |
| UChatTool         | 工具調用狀態（inline / card 模式）  |
| UChatPrompt       | 增強版 Textarea，支援 error 顯示    |
| UChatPromptSubmit | 送出按鈕，自動處理 send/stop/reload |
| UChatPalette      | 在 Modal/Slideover 內使用的佈局容器 |

**Modal 內的 Chat**

```vue
<UModal v-model:open="isOpen">
  <template #content>
    <UChatPalette>
      <UChatMessages :messages="chat.messages" :status="chat.status" />

      <template #prompt>
        <UChatPrompt v-model="input" @submit="onSubmit">
          <UChatPromptSubmit :status="chat.status" />
        </UChatPrompt>
      </template>
    </UChatPalette>
  </template>
</UModal>
```

**帶模型選擇器**

```vue
<UChatPrompt v-model="input" @submit="onSubmit">
  <UChatPromptSubmit :status="chat.status" />

  <template #footer>
    <USelect
      v-model="model"
      :icon="models.find(m => m.value === model)?.icon"
      placeholder="選擇模型"
      variant="ghost"
      :items="models"
    />
  </template>
</UChatPrompt>
```

## Do's and Don'ts

### Do

- 用字重和大小區分層級
- 用邊框和陰影區分元件邊界
- 保持大量留白
- 對話訊息用圓角區分使用者/AI
- 專注於內容可讀性

### Don't

- 不使用彩色強調按鈕（除了語意色彩的 alert）
- 不使用漸層背景
- 不使用過多 icon 裝飾
- 不使用動畫（除了必要的載入指示）
- 不使用陰影層疊（最多單層淺陰影）

---

## Accessibility

- **對比度**：黑白配色天然達到 WCAG AAA
- **字體大小**：最小 14px（sm），一般 16px（base）
- **焦點狀態**：使用 Nuxt UI 元件內建焦點樣式，或 `focus:ring-2 focus:ring-accented`
- **減少動態**：尊重 `prefers-reduced-motion`
