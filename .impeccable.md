# Design System

## Design Context

### Users

企業內部員工，在日常工作中使用此 RAG 應用查詢知識庫、搜尋內部文件、獲得工作輔助。使用情境通常是：

- 快速查找公司政策、流程文件
- 詢問技術或業務相關問題
- 需要專注且高效的工作輔助工具

### Brand Personality

**三個關鍵詞**：簡約、高效、可靠

- **語氣**：專業但不冷漠，直接但不粗魯
- **情感目標**：讓使用者感到工具值得信賴、能快速完成任務
- **互動風格**：最少點擊、最快到達答案

### Aesthetic Direction

**視覺基調**：純黑白極簡主義

- **配色**：僅使用黑、白、灰階（neutral palette）
- **無彩色強調**：不使用 primary color，改用字重、大小、對比度區分層級
- **參考**：ChatGPT / Claude 的對話式介面——乾淨的訊息流、簡潔的輸入區、專注於內容
- **Anti-reference**：避免花俏的漸層、彩色按鈕、過多裝飾元素

**主題**：Light mode 為主（企業環境常見），可選支援 dark mode

### Design Principles

1. **內容優先**：UI 應該隱形，讓使用者專注於對話內容和搜尋結果
2. **一眼可辨**：資訊層級清晰，重要操作一目了然
3. **最短路徑**：減少點擊次數，常用功能觸手可及
4. **一致性**：相同功能用相同樣式，不製造認知負擔
5. **無障礙優先**：高對比度黑白配色天然利於閱讀

## Color Tokens

基於 Nuxt UI / Tailwind 4，使用純黑白作為 primary：

```css
/* app/assets/css/main.css */
:root {
  --ui-radius: 0.375rem;
  --ui-primary: black;
}

.dark {
  --ui-primary: white;
}
```

- Light mode：primary 為純黑
- Dark mode：primary 為純白
- 圓角統一 6px (0.375rem)

**語意色彩**（僅用於系統回饋）：

| 用途 | 顏色        | 使用時機               |
| ---- | ----------- | ---------------------- |
| 錯誤 | red-500     | 表單驗證、系統錯誤     |
| 警告 | amber-500   | 重要提醒、風險操作確認 |
| 成功 | green-500   | 操作完成、儲存成功     |
| 資訊 | neutral-500 | 一般提示               |

**重要**：語意色彩僅用於系統回饋（alert、toast），不用於裝飾或強調。

## Typography

使用 DM Sans 作為主要字體：

```css
@theme {
  --font-sans: "DM Sans", sans-serif;
}
```

DM Sans 特性：

- 幾何無襯線字體，清晰現代
- 良好的中英文混排效果
- Nuxt Fonts 自動載入

**層級區分**（用字重而非顏色）：

| 層級 | 字重 | 大小 | 用途               |
| ---- | ---- | ---- | ------------------ |
| H1   | 700  | 2xl  | 頁面標題           |
| H2   | 600  | xl   | 區塊標題           |
| H3   | 600  | lg   | 子區塊標題         |
| Body | 400  | base | 內文               |
| Meta | 400  | sm   | 次要資訊、時間戳記 |
| Code | 400  | sm   | 程式碼、技術名詞   |

## Spacing

採用 Tailwind 4 預設間距系統，保持節奏一致：

| Token | 值   | 用途                       |
| ----- | ---- | -------------------------- |
| 1     | 4px  | 最小間距                   |
| 2     | 8px  | 緊湊元素間                 |
| 3     | 12px | 相關元素間                 |
| 4     | 16px | 表單欄位間、卡片內 padding |
| 6     | 24px | 區塊間                     |
| 8     | 32px | 主要區塊間                 |

## Component Patterns

### 對話式介面（參考 ChatGPT/Claude）

```vue
<!-- 訊息容器 -->
<div class="flex flex-col gap-4">
  <!-- 使用者訊息 -->
  <div class="flex justify-end">
    <div class="max-w-[80%] rounded-2xl rounded-br-md bg-neutral-100 px-4 py-3">
      使用者輸入內容
    </div>
  </div>

  <!-- AI 回應 -->
  <div class="flex justify-start">
    <div class="max-w-[80%] rounded-2xl rounded-bl-md bg-white border border-neutral-200 px-4 py-3">
      AI 回應內容
    </div>
  </div>
</div>

<!-- 輸入區 -->
<div class="border-t bg-white p-4">
  <div class="flex items-end gap-2">
    <UTextarea placeholder="輸入問題..." autoresize :rows="1" class="flex-1" />
    <UButton color="neutral" variant="solid" icon="i-lucide-send" />
  </div>
</div>
```

### 按鈕

純黑白系統中，按鈕用 neutral 色階：

```vue
<!-- Primary action -->
<UButton color="neutral" variant="solid">確認</UButton>

<!-- Secondary action -->
<UButton color="neutral" variant="outline">取消</UButton>

<!-- Ghost action -->
<UButton color="neutral" variant="ghost">更多選項</UButton>
```

### 表單

```vue
<UFormField label="欄位名稱">
  <UInput placeholder="請輸入..." />
</UFormField>
```

### 空狀態

```vue
<div class="flex flex-col items-center justify-center py-12 text-center">
  <div class="text-neutral-400 mb-2">
    <UIcon name="i-lucide-inbox" class="size-12" />
  </div>
  <p class="text-neutral-600 font-medium">尚無資料</p>
  <p class="text-neutral-400 text-sm mt-1">描述如何開始或建立第一筆資料</p>
</div>
```

## Nuxt UI Configuration

**app/app.config.ts**：

```typescript
export default defineAppConfig({
  ui: {
    colors: {
      neutral: "neutral",
    },
  },
});
```

**app/assets/css/main.css**：

```css
@import "tailwindcss";
@import "@nuxt/ui";

@theme {
  --font-sans: "DM Sans", sans-serif;
}

:root {
  --ui-radius: 0.375rem;
  --ui-primary: black;
}

.dark {
  --ui-primary: white;
}
```

- 不設定 primary color palette，改用 CSS 變數直接指定黑/白
- neutral 使用 Tailwind 原生 neutral（非 zinc）

## Do's and Don'ts

### Do

- 用字重和大小區分層級
- 用邊框和陰影區分元件邊界
- 保持大量留白
- 對話訊息用圓角區分使用者/AI
- 專注於內容可讀性

### Don't

- 不使用彩色強調按鈕（除了語意色彩的 alert）
- 不使用漸層背景
- 不使用過多 icon 裝飾
- 不使用動畫（除了必要的載入指示）
- 不使用陰影層疊（最多單層淺陰影）

---

## Accessibility

- **對比度**：黑白配色天然達到 WCAG AAA
- **字體大小**：最小 14px（sm），一般 16px（base）
- **焦點狀態**：使用 `focus:ring-2 focus:ring-neutral-400`
- **減少動態**：尊重 `prefers-reduced-motion`
